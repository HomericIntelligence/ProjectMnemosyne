# Ruff Exclude vs Per-File-Ignores for Generated Files

## Overview

| Field | Value |
|-------|-------|
| **Date** | 2026-04-21 |
| **Objective** | Suppress all ruff processing (lint + format) for a generated file such as `hephaestus/_version.py` produced by `hatch-vcs` |
| **Outcome** | Success — `ruff format --check` passes after moving from `per-file-ignores` to top-level `exclude` |
| **Verification** | verified-precommit — confirmed locally that `ruff format --check` passes; CI validation pending |

## When to Use

- A generated file (e.g., `_version.py` from `hatch-vcs`, `*_pb2.py` from protobuf, `parser.py` from PLY/ANTLR) causes `ruff format --check` failures in CI
- You already added the file to `[tool.ruff.lint.per-file-ignores]` with `["ALL"]` but CI still fails on `ruff format`
- You want to completely exclude a file from **both** `ruff check` and `ruff format`
- A `.gitignore` entry does not stop ruff from processing a file that exists on disk at lint time

## Verified Workflow

### Quick Reference

```toml
# pyproject.toml

[tool.ruff]
line-length = 100
target-version = "py310"
# Exclude generated files from BOTH ruff check AND ruff format:
exclude = ["hephaestus/_version.py"]

# Do NOT add generated files here — per-file-ignores only suppresses lint rules,
# it has no effect on ruff format --check:
[tool.ruff.lint.per-file-ignores]
"tests/**" = ["S101", "D102", "D107"]
```

### Detailed Steps

1. Identify the generated file causing failures. Check the CI `ruff format --check` output for the filename.

2. Search for the file in `[tool.ruff.lint.per-file-ignores]` in your `pyproject.toml`. If it is there with `["ALL"]`, remove it.

3. Add the file path to the top-level `[tool.ruff]` section under `exclude`:

   ```toml
   [tool.ruff]
   exclude = ["path/to/generated_file.py"]
   ```

4. Verify locally:

   ```bash
   ruff format --check .
   ruff check .
   pre-commit run ruff-format --all-files
   pre-commit run ruff --all-files
   ```

5. Commit and push. CI should now pass.

## Failed Attempts

| Attempt | What Was Tried | Why It Failed | Lesson Learned |
|---------|----------------|---------------|----------------|
| Attempt 1 | Added `"hephaestus/_version.py" = ["ALL"]` to `[tool.ruff.lint.per-file-ignores]` | Silenced `ruff check` violations but `ruff format --check` still flagged the file because `per-file-ignores` only applies to the `check` subcommand, not to `format` | `per-file-ignores` is scoped to lint rules only; it is under `[tool.ruff.lint]` for a reason |
| Attempt 2 | Assumed `.gitignore` entry would prevent ruff from scanning the file | `.gitignore` has no effect on ruff; ruff scans whatever files exist on disk regardless of git tracking status | Generated files that exist at lint time must be explicitly excluded in `pyproject.toml` |

## Results & Parameters

### Key Distinction

| Config | Affects `ruff check` | Affects `ruff format` |
|--------|---------------------|----------------------|
| `[tool.ruff.lint.per-file-ignores]` | Yes — suppresses specific lint rules | **No** |
| `[tool.ruff] exclude` | Yes — skips file entirely | **Yes — skips file entirely** |

### When to Use Each

- **`[tool.ruff] exclude`** — For generated files that should be entirely ignored (both lint and format). Examples: `_version.py`, `*_pb2.py`, parser output.
- **`[tool.ruff.lint.per-file-ignores]`** — For source files where you want to suppress specific lint rules only (e.g., allow `assert` in tests via `S101`). Does NOT affect formatting.

### Example: hatch-vcs Generated Version File

```toml
[tool.ruff]
line-length = 100
target-version = "py310"
exclude = ["hephaestus/_version.py"]   # Generated by hatch-vcs at install time

[tool.ruff.lint]
select = ["E", "F", "W", "I", "N", "D", "UP"]
ignore = ["D100", "D104", "D203", "D213"]

[tool.ruff.lint.per-file-ignores]
"tests/**" = ["S101", "D102", "D107"]  # test-specific suppressions only
```

### Note on `.gitignore`

Adding a generated file to `.gitignore` prevents it from being tracked by git, but ruff ignores `.gitignore` entirely. If the file exists on disk when `ruff format --check` runs (e.g., after `pip install -e .` or `hatch build`), ruff will process it unless it is explicitly listed in `[tool.ruff] exclude`.
