# Per-Module Coverage Floor Enforcement

## Overview

| Field | Value |
|-------|-------|
| **Date** | 2026-05-28 |
| **Objective** | Enforce minimum coverage thresholds at per-file granularity, preventing aggregate gates from masking under-tested individual modules |
| **Outcome** | Successfully implemented in ProjectHephaestus issue #623; enforces branch_rate > 0 else line_rate comparison |
| **Verification** | verified-local (48 unit tests pass; CI validation pending on merge) |

## When to Use

- You have an aggregate coverage gate (e.g., 85%) but want per-file minimums (e.g., schema.py ≥ 80%)
- You're adding a new module and want to enforce its minimum floor immediately
- You've noticed that aggregate coverage gate is masking under-tested code in specific files
- You want CI to fail loudly if a configured module is absent from coverage report (regression signal)

## Verified Workflow

### Quick Reference

```bash
# 1. Add parse_module_coverage() to hephaestus/validation/coverage.py
# Parses Cobertura XML <class> elements to extract per-file coverage

# 2. Create coverage.toml in repo root
cat > coverage.toml << 'EOF'
[coverage.modules."hephaestus/validation/schema.py"]
minimum = 80

[coverage.modules."hephaestus/cli/main.py"]
minimum = 85
EOF

# 3. Run full test suite
pytest tests/unit -v

# 4. Run CI step: check-coverage against coverage.toml
# (runs after pytest generates .coverage + coverage.xml)
hephaestus-check-coverage --config coverage.toml

# Expected: PASS if all configured modules meet floor
# Expected: FAIL (exit 1) if module missing from report or below floor
```

### Detailed Steps

1. **Implement parse_module_coverage() function** in `hephaestus/validation/coverage.py`:
   - Reads Cobertura XML (generated by pytest-cov)
   - Iterates `<class>` elements (each is a file)
   - Extracts `filename`, `branch_rate`, `line_rate`
   - Returns dict: `{filename: {"branch_rate": float, "line_rate": float}}`
   - Use `branch_rate > 0 else line_rate` for comparison (handles files with no branches)

2. **Create coverage.toml** in repository root:
   ```toml
   [coverage.modules."path/to/file.py"]
   minimum = 80
   ```
   - One `[coverage.modules."path"]` per configured module
   - `minimum` is the floor (integer 0-100)
   - CI step validates all configured modules are present + above floor

3. **Create hephaestus-check-coverage CLI command** (or extend existing):
   - Reads coverage.toml + generated coverage.xml
   - Calls parse_module_coverage() to extract per-file rates
   - For each `[coverage.modules]` entry in .toml:
     - Check if file exists in coverage report
     - Compare (branch_rate > 0 else line_rate) to minimum
     - Print PASS/FAIL for each module
   - Exit 1 if any module missing or below floor
   - Exit 0 if all pass

4. **Add CI step** (after pytest runs):
   ```yaml
   - name: Check per-module coverage floors
     run: hephaestus-check-coverage --config coverage.toml
   ```

## Failed Attempts

| Attempt | What Was Tried | Why It Failed | Lesson Learned |
|---------|----------------|---------------|----------------|
| Using parse_coverage_report() for per-module | Reused existing function that only returns aggregate {line: X, branch: Y} | Function was designed for repo-wide stats; returns single dict, not per-file breakdown | Always check function return type before reusing — per-file logic requires dict-of-dicts |
| Running check as unit test instead of CI step | Wrote `test_coverage_floors()` that reads coverage.xml and validates floors | Unit test ran only on full test suite; CI could run partial tests (unit-only, fast subset) → false pass | CI step must run after *full* test suite to catch per-module skips; unit tests are insufficient |
| Using line_rate for all files | Assumed line_rate was always available/relevant | Files with no branches (only if/imports) have line_rate=100 but branch_rate=0 → false pass for untested branches | Use `branch_rate > 0 else line_rate` — prioritize branch coverage, fall back to line coverage |

## Results & Parameters

### Configuration Example

```toml
# coverage.toml — Minimum per-module floors
[coverage.modules."hephaestus/validation/schema.py"]
minimum = 80

[coverage.modules."hephaestus/cli/main.py"]
minimum = 85

[coverage.modules."hephaestus/logging/__init__.py"]
minimum = 90
```

### Expected Output

```
Checking per-module coverage floors...

✓ hephaestus/validation/schema.py: 94.81% (minimum 80%)
✓ hephaestus/cli/main.py: 87.2% (minimum 85%)
✓ hephaestus/logging/__init__.py: 91.5% (minimum 90%)

All modules passed. Exit code 0.
```

```
Checking per-module coverage floors...

✗ hephaestus/validation/schema.py: Missing from coverage report
✗ hephaestus/cli/main.py: 82.1% (minimum 85%) — BELOW FLOOR

2 modules failed. Exit code 1.
```

### Key Parameters

- **branch_rate vs line_rate**: Always use `max(branch_rate, 0) or line_rate` — branch coverage is stricter
- **Cobertura XML path**: Default `.coverage.xml` (pytest-cov output)
- **coverage.toml path**: Default `./coverage.toml` in repo root, override with `--config`
