# self-cancelling-review-plan

Handle `.claude-review-fix-*.md` files generated by review automation where the outer
wrapper instructs "implement all fixes" but the inner Fix Plan body concludes no changes
are required.

## Overview

| Item | Details |
| ------ | --------- |
| Date | 2026-03-05 |
| Issue | #3152 |
| PR | #3343 |
| Objective | Implement review fixes for a Docker PR |
| Outcome | No-op — plan declared PR already correct; confirmed with git status |
| Category | ci-cd |
| Project | ProjectOdyssey |

## When to Use

- You receive a `.claude-review-fix-<number>.md` file with the instruction "implement all fixes"
- The embedded **Fix Plan** section states the PR is correct and complete with "No fixes required"
- The failing CI jobs listed are pre-existing failures on `main` unrelated to the PR's scope
- `git status` shows a clean working tree (no uncommitted changes)
- The only untracked file is the review instructions file itself

## Verified Workflow

1. **Read the review file** — parse both the outer wrapper instruction ("implement all fixes")
   and the inner Fix Plan body carefully
2. **Check the Fix Order section** — if it says "No fixes are required" or equivalent, the
   plan is self-cancelling
3. **Verify git status** — run `git status` and `git log --oneline -5` to confirm the fix
   commit already exists on the branch
4. **Do not manufacture commits** — if the branch is already correct, make no changes
5. **Report clearly** — explain that the fix plan declares no action needed and the branch
   is already up to date

```bash
# Step 3: Confirm branch state
git status
git log --oneline -5

# Expected: clean working tree, fix commit present, only .claude-review-fix-*.md untracked
```

## Key Signals That a Plan Is Self-Cancelling

| Signal | Location in File | Meaning |
| -------- | ----------------- | --------- |
| "No fixes are required" | Fix Order section | Plan explicitly cancels itself |
| "The PR is correct and complete" | Summary section | Implementation already done |
| "Pre-existing failures on main" | Problems Found section | Failing CI is not caused by PR |
| All checklist items marked ✅ | Fix Order list | Everything already implemented |
| Clean `git status` | Terminal | No uncommitted changes exist |

| Attempt | What Was Tried | Why It Failed | Lesson Learned |
| --------- | ---------------- | --------------- | ---------------- |
| N/A — correct approach taken immediately | Read the full fix plan before acting | — | Always read the entire plan body, not just the outer wrapper instruction |

### Self-Cancelling Pattern Template

A review fix file is self-cancelling when it matches this structure:

```text
Implement the fixes described in the plan below for PR #<N> (issue #<M>).

**Fix Plan:**
<analysis text>

## Fix Order

No fixes are required. The PR is correct and complete:
1. <change> ✅
2. <change> ✅
...
```

The outer shell says "implement fixes." The inner plan says "fixes = nothing." Trust the inner plan.
