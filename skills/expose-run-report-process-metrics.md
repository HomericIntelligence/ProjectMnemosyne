# expose-run-report-process-metrics

How to add a new metrics section to the per-run `report.md` and `report.json` files generated by
`stage_write_report()` in ProjectScylla, using process metrics (R_Prog, Strategic Drift, CFP,
PR Revert Rate) as the concrete example.

## When to Use

- A new metrics block already exists in `run_result.json` (written by `stage_finalize_run`) and
  you want to include it in the per-run report outputs
- Adding a `## New Section` to the markdown generated by `generate_run_report()`
- Adding a new key to the structured JSON produced by `save_run_report_json()`
- Any change where `stage_write_report()` needs to read additional data from disk and pass it to
  the report functions

## Architecture: Per-Run Report Pipeline

```
stage_finalize_run()          → writes run_result.json (includes process_metrics block)
    ↓
stage_write_report()          → reads run_result.json, calls save_run_report() + save_run_report_json()
    ↓
save_run_report()             → calls generate_run_report() → writes report.md
save_run_report_json()        → writes report.json
```

Key insight: `run_result.json` is the authoritative source written by the previous stage.
`stage_write_report()` reads it via `json.load()` with a `try/except (OSError, json.JSONDecodeError)` guard.

## Verified Workflow

### Step 1: Add helper functions to `run_report.py`

Add a value formatter and a section generator before `generate_run_report()`:

```python
import math  # add to imports

def _format_process_metric_value(val: Any) -> str:
    """Format a process metric value, returning 'N/A' for None/NaN."""
    if val is None:
        return "N/A"
    try:
        f = float(val)
        if math.isnan(f):
            return "N/A"
        return f"{f:.4f}"
    except (TypeError, ValueError):
        return "N/A"


def _generate_process_metrics_section(process_metrics: dict[str, Any]) -> list[str]:
    """Generate ## Process Metrics markdown table."""
    r_prog = _format_process_metric_value(process_metrics.get("r_prog"))
    strategic_drift = _format_process_metric_value(process_metrics.get("strategic_drift"))
    cfp = _format_process_metric_value(process_metrics.get("cfp"))
    pr_revert_rate = _format_process_metric_value(process_metrics.get("pr_revert_rate"))
    return [
        "---", "",
        "## Process Metrics", "",
        "| Metric | Value |",
        "|--------|-------|",
        f"| R_Prog (Fine-Grained Progress) | {r_prog} |",
        f"| Strategic Drift | {strategic_drift} |",
        f"| CFP (Change Fail %) | {cfp} |",
        f"| PR Revert Rate | {pr_revert_rate} |",
        "",
    ]
```

### Step 4: Add NaN-guarded JSON serialization to `save_run_report_json()`

JSON does not support `NaN`/`Infinity` literals — guard before writing:

```python
def save_run_report_json(
    ...
    process_metrics: dict[str, Any] | None = None,
) -> None:
    report: dict[str, Any] = {
        "run_number": run_number, "score": score, ...
    }
    if process_metrics and isinstance(process_metrics, dict):
        guarded: dict[str, float | None] = {}
        for key in ("r_prog", "strategic_drift", "cfp", "pr_revert_rate"):
            val = process_metrics.get(key)
            if val is None:
                guarded[key] = None
            else:
                try:
                    f = float(val)
                    guarded[key] = None if math.isnan(f) else f
                except (TypeError, ValueError):
                    guarded[key] = None
        report["process_metrics"] = guarded
    (run_dir / "report.json").write_text(json.dumps(report, indent=2))
```

### Step 5: Update `stage_write_report()` in `stages.py`

Read the metrics from the already-written `run_result.json` (safe — it was written by the
immediately preceding `stage_finalize_run()` in the same state-machine run):

```python
# Read process_metrics from run_result.json written by stage_finalize_run
process_metrics: dict[str, Any] | None = None
try:
    with open(ctx.run_dir / "run_result.json") as f:
        run_result_data = json.load(f)
    process_metrics = run_result_data.get("process_metrics")
except (OSError, json.JSONDecodeError, KeyError):
    pass

save_run_report(
    ...
    process_metrics=process_metrics,
)
save_run_report_json(
    ...
    process_metrics=process_metrics,
)
```

`json` and `Any` are already imported in `stages.py`.

## Key Gotchas

1. **NaN in JSON**: Python's `json.dumps()` will serialize `float("nan")` as `NaN` by default,
   which is invalid JSON. Always guard with `math.isnan()` and replace with `None`.

2. **`math` must be explicitly imported** in `run_report.py` even though it is a stdlib module.

3. **Empty dict guard**: `if process_metrics and isinstance(...)` correctly skips empty dicts
   (falsy in Python), preventing empty `## Process Metrics` sections.

4. **Reading from `run_result.json`**: Do NOT read `process_metrics` from `ctx.run_result` —
   the `E2ERunResult` dataclass does not carry these fields. They are only in the JSON file
   written by `stage_finalize_run()`.

5. **Mypy in tests**: Helper methods that build keyword-argument dicts must use `dict[str, Any]`
   return type, not bare `dict`, to pass mypy strict type checking.
