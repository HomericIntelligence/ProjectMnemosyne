### Issue
GitHub issue #1164: `test_manage_experiment.py` (4002+ lines, grew to 4212) hits Edit tool
token limit. Filed as follow-up to #1113 (close script test gap).

### Problem Statement
The Edit tool requires a prior `Read` call to succeed. Files exceeding ~25K tokens cause
both tools to fail (Read returns truncated content; Edit can't find old_string matches).
The `bash cat-append` workaround works but is fragile and hard to review.

### Key Technical Decisions

1. **Python script extraction** (not manual copy-paste): used `splitlines(keepends=True)` +
   slice by 0-indexed section boundaries computed from separator comment positions.

2. **Section boundary algorithm**: walk backwards from class definition to find the
   `# ---` separator comment block. Pattern is always 5 lines: blank + `# ---` + `# Title`
   + `# ---` + blank.

3. **`find_section_start()` function**: walks back past blank lines, then past `#` comment
   lines, then returns i+1. Reliably identifies the separator start for all 40 classes.

4. **Ruff auto-fixed on first run**: 30 errors (unused imports in new file headers, minor
   formatting). Expected — ruff modifies files then reports "files modified". Run twice.
