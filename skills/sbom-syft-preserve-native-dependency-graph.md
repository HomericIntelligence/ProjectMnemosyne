# Preserve Syft's Native Dependency Graph in Deterministic SBOMs

## Overview

| Field | Value |
| ------- | ------- |
| **Date** | 2026-07-15 |
| **Objective** | Produce byte-reproducible SPDX 2.3 release evidence while retaining Syft's package, file, and dependency relationships. |
| **Outcome** | Athena PR #14 generated byte-identical consecutive SBOM/checksum pairs in required Linux CI while preserving `CONTAINS`, `DEPENDS_ON`, `DEPENDENCY_OF`, and `OTHER` relationships. |
| **Verification** | verified-ci |

## When to Use

- Post-processing Syft SPDX JSON to remove timestamps, temporary paths, or random namespaces.
- Publishing separate SBOMs for a release archive and its authoritative build environment.
- A deterministic rewrite unexpectedly collapses the SBOM into only package/file containment.
- CI must prove SBOM and checksum bytes are reproducible on the release platform.
- Dependency discovery emits ambiguous guesses whose order or target changes across runs.

## Verified Workflow

### Quick Reference

```bash
# Generate twice in the same locked Linux job, using distinct output directories.
python3 scripts/generate_sboms.py --output dist
python3 scripts/generate_sboms.py --output dist-reproducibility

# Compare both SPDX documents and their checksum sidecars byte for byte.
cmp dist/athena-plugin-*.spdx.json dist-reproducibility/athena-plugin-*.spdx.json
cmp dist/athena-plugin-*.spdx.json.sha256 dist-reproducibility/athena-plugin-*.spdx.json.sha256
cmp dist/athena-build-linux-64-*.spdx.json dist-reproducibility/athena-build-linux-64-*.spdx.json
cmp dist/athena-build-linux-64-*.spdx.json.sha256 dist-reproducibility/athena-build-linux-64-*.spdx.json.sha256
```

### Detailed Steps

1. Lock Syft and the build environment. Run generation on the same platform used to produce the
   release artifact; platform-specific package inventories should say so in their filenames.
2. Ask Syft to produce the native SPDX document. Treat its packages, files, checksums, external
   references, and relationships as semantic evidence—not as disposable input for a new document.
3. Canonicalize only known nondeterministic fields: creation timestamps, random document
   namespaces, host-specific absolute paths, and ordering where the order has no meaning.
4. Preserve native relationship records and referenced SPDX identifiers. If an ambiguous
   dependency guess cannot be mapped deterministically to one target, omit that guess instead of
   inventing a different edge. Do not replace the native graph with a handcrafted containment graph.
5. Give each document a deterministic namespace derived from stable content and release identity.
   Keep the release package identity and version explicit so the publisher can bind assets to the
   intended version.
6. Serialize canonical JSON consistently and emit a checksum sidecar that names the exact artifact.
7. In one required Linux CI job, generate the complete set twice and compare every document and
   checksum byte for byte. A unit test of the normalization function is useful but is not an
   end-to-end reproducibility proof.
8. Before publication, verify the exact release asset set, each checksum pair, SPDX 2.3 identity,
   namespace, and matching release package/version. Test rejection of missing, extra, renamed,
   corrupted, or version-mismatched assets.

## Failed Attempts

| Attempt | What Was Tried | Why It Failed | Lesson Learned |
| ------- | -------------- | ------------- | -------------- |
| Rebuilt SPDX from a simplified package/file model | Post-processing constructed new `CONTAINS` relationships and discarded the Syft graph | Dependency edges such as `DEPENDS_ON` and `DEPENDENCY_OF` vanished even though the JSON remained valid | Mutate only nondeterministic fields in the native document; preserve semantic records and IDs. |
| Sorted ambiguous dependency guesses | Multiple packages could satisfy a guessed dependency, so sorting merely chose a stable but potentially wrong target | Determinism was achieved by fabricating semantics | Retain verified native edges; omit an ambiguous inferred edge when no deterministic mapping exists. |
| Proved determinism only in unit tests | Normalizer fixtures passed locally | The real Syft binary, locked environment, filesystem inventory, and Linux workflow were untested | Generate the complete artifact set twice in required CI and compare bytes. |
| Verified checksums but not release binding | Each sidecar matched its file | A valid SBOM for the wrong product or version could still be published | Verify the exact six-file set plus SPDX name, namespace, package, and version. |
| Let every required job upload release evidence | Multiple jobs gained artifact-write capability | The permission surface exceeded the job that actually built evidence | Scope artifact upload actions and permissions to the package job only. |
