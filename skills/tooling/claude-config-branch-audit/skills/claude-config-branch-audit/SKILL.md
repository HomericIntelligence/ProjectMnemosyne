## Overview

| Field | Value |
|-------|-------|
| **Skill** | claude-config-branch-audit |
| **Category** | tooling |
| **Applies To** | Any repo with Claude Code configuration in CLAUDE.md or .claude/ |
| **Trigger** | Branch adds/modifies Claude Code configuration files |

## When to Use

- A feature branch (e.g. `claude-configuration-optimization`) adds or rewrites `CLAUDE.md`
- `.claude/settings.json` references hook scripts that don't exist in the repository
- CLAUDE.md references a Python version, tooling stack (pip/venv/flake8/black/tox), or directory structure that doesn't match the repo's actual setup
- Merging a "Claude config optimization" branch generated by an external agent or another repo

## Verified Workflow

### Step 1 — Identify what changed on the branch

```bash
git diff origin/main...<branch> --stat
git log --oneline origin/main...<branch>
```

Focus on: `CLAUDE.md`, `.claude/settings.json`, any new `.claude/` subdirectories.

### Step 2 — Check for dangling hook references

```bash
git show <branch>:.claude/settings.json
```

For every `"command"` referencing a `.claude/hooks/*.py` script, verify the file exists:

```bash
git show <branch>:.claude/hooks/<script>.py 2>/dev/null || echo "MISSING"
```

If missing → remove the hook entry from `settings.json`.

### Step 3 — Verify Python version

Compare the Python version stated in `CLAUDE.md` against:
- `pyproject.toml` `requires-python`
- `pixi.toml` Python constraint
- CI matrix (`.github/workflows/*.yml`)

Update `CLAUDE.md` to match (all occurrences — typically "Language Preference" and "Requirements" sections).

### Step 4 — Verify tooling commands

Check which environment manager and linter the repo actually uses:

```bash
ls pyproject.toml pixi.toml requirements.txt 2>/dev/null
```

| If repo uses | Replace CLAUDE.md references to |
|---|---|
| `pixi` | `pip`, `venv`, `tox` |
| `ruff` | `flake8`, `black`, `isort` |
| `pixi run pytest tests/unit` | `python -m pytest tests/` |

### Step 5 — Verify repository structure

Cross-check the directory tree listed in `CLAUDE.md` against actual `hephaestus/` (or equivalent) subdirectories:

```bash
ls hephaestus/
```

Remove modules that don't exist; add modules that are present but missing from the doc.

### Step 6 — Fix key files section

Ensure `CLAUDE.md`'s "Key Files and Directories" section references the actual dependency manifests (`pyproject.toml`, `pixi.toml`) rather than legacy ones (`requirements.txt`, `requirements-dev.txt`).

### Step 7 — Commit fixes

```bash
git add CLAUDE.md .claude/settings.json
git commit -m "fix(claude): <summary of fixes>"
```

## Failed Attempts

| Attempt | What Was Tried | Why It Failed | Lesson Learned |
|---------|----------------|---------------|----------------|
| Merging branch as-is | Accepting the configuration-optimization branch without review | hooks referenced non-existent scripts; would break every Bash tool call via PreToolUse hook | Always verify hook script paths exist before merging .claude/settings.json changes |
| Trusting generated Python version | Branch stated Python 3.8+ (likely copied from a template) | Repo requires 3.12+; stale version creates confusion for contributors | Cross-check Python version in CLAUDE.md against pyproject.toml and CI matrix |
| Trusting generated tooling commands | Branch used `pip`, `flake8`, `black`, `tox` | Repo uses `pixi` + `ruff`; commands would silently fail for new contributors | Always verify environment manager and linter by checking pyproject.toml/pixi.toml |

## Results & Parameters

### settings.json — safe minimal config (no hooks)

```json
{
  "enabledPlugins": {
    "skills-registry-commands@ProjectMnemosyne": true,
    "safety-net@cc-marketplace": true
  }
}
```

Only add hooks when the referenced scripts are committed to the same branch.

### CLAUDE.md tooling block — pixi + ruff pattern

```bash
# Install dependencies and create environment
pixi install

# Run pre-commit hooks (formatting, linting, etc.)
pre-commit install
```

```bash
# Run tests
pixi run pytest tests/unit

# Run linter
pixi run ruff check hephaestus/ tests/

# Run formatter
pixi run ruff format hephaestus/ tests/

# Run type checking
pixi run mypy hephaestus/
```
