# CI Workflow Missing Dependency Install

## Overview

| Field | Value |
|-------|-------|
| **Date** | 2026-03-27 |
| **Objective** | Fix failing Update Marketplace CI workflow on main branch |
| **Outcome** | Successful - one-line fix resolved all failures |

## When to Use

- A GitHub Actions workflow fails with `ModuleNotFoundError: No module named '<package>'`
- Multiple workflows share a Python script but only some install the required dependencies
- CI worked previously but broke when a new Python import was added to a shared script
- Local runs succeed (because the package is installed locally) but CI fails

## Verified Workflow

### Quick Reference

```yaml
# Add before the step that runs the Python script:
- name: Install dependencies
  run: pip install <missing-package>
```

### Detailed Steps

1. **Identify the failure**: Run `gh run list --branch main --limit 5` to find failing runs
2. **Get error logs**: Run `gh run view <run-id> --log-failed` to see the actual error
3. **Find the root cause**: Look for `ModuleNotFoundError` and identify the missing package
4. **Check sibling workflows**: Compare with other workflows that run similar scripts - they likely already install the dependency
5. **Add the install step**: Add `pip install <package>` between "Set up Python" and the script execution step
6. **Verify locally**: Run the script locally to confirm it works
7. **Submit via PR**: Create a branch, commit, push, and open a PR

## Failed Attempts

| Attempt | What Was Tried | Why It Failed | Lesson Learned |
|---------|----------------|---------------|----------------|
| Guessing the fix without reading logs | Assumed the issue was in the script itself | The script was correct; the workflow was missing a dependency step | Always read `gh run view --log-failed` output before attempting fixes |
| Committing regenerated marketplace.json alongside the fix | Ran the script locally which regenerated the output file | The marketplace.json is supposed to be generated by CI, not committed manually | Only commit the fix, not side effects of local testing |

## Results & Parameters

**Before fix** (`update-marketplace.yml`):
```yaml
steps:
  - name: Checkout code
    uses: actions/checkout@v4
  - name: Set up Python
    uses: actions/setup-python@v5
    with:
      python-version: '3.11'
  - name: Generate marketplace.json  # <- fails here
    run: python3 scripts/generate_marketplace.py
```

**After fix**:
```yaml
steps:
  - name: Checkout code
    uses: actions/checkout@v4
  - name: Set up Python
    uses: actions/setup-python@v5
    with:
      python-version: '3.11'
  - name: Install dependencies  # <- added this step
    run: pip install pyyaml
  - name: Generate marketplace.json
    run: python3 scripts/generate_marketplace.py
```

**Pattern to check**: When adding Python imports to shared scripts, grep all workflows that call that script:
```bash
grep -rl "script_name.py" .github/workflows/
```
Then verify each workflow installs the required packages.
