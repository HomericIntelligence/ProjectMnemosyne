# Conan Install Missing in GitHub Actions CI

## Overview

| Field | Value |
|-------|-------|
| **Date** | 2026-04-07 |
| **Objective** | Fix GitHub Actions CI failing at `cmake --preset` because `conan_toolchain.cmake` was never generated |
| **Outcome** | All 4 matrix builds (gcc/clang × debug/release) pass CI after adding conan install step |
| **Verification** | verified-ci |

## When to Use

- CI fails at cmake configure step with "Could not find toolchain file: build/*/conan_toolchain.cmake"
- Adding CI to a C++ project that uses CMakePresets.json with Conan toolchain
- Existing CI installs `ninja-build` but has no conan step
- All three workflow types (build-test, static-analysis, code-coverage) fail at configure
- Project uses `CMAKE_TOOLCHAIN_FILE` pointing to a Conan-generated path

## Verified Workflow

### Quick Reference

```yaml
# Add these steps before "Configure" in any GitHub Actions CI job:
- name: Install dependencies
  run: |
    sudo apt-get install -y ninja-build
    pip install conan
    conan profile detect --force

# For matrix builds with dynamic build_type (debug/release):
- name: Install Conan dependencies
  env:
    BUILD_TYPE: ${{ matrix.build_type }}
  run: |
    BUILD_TYPE_CAP=$(echo "$BUILD_TYPE" | sed 's/./\u&/')
    conan install . --build=missing -s build_type="$BUILD_TYPE_CAP" \
      --output-folder "build/$BUILD_TYPE"

# For fixed build type (static-analysis/coverage):
- name: Install Conan dependencies
  run: |
    conan install . --build=missing -s build_type=Debug \
      --output-folder build/debug
```

### Detailed Steps

1. In the "Install dependencies" step, add `pip install conan` and `conan profile detect --force` after apt-get installs
2. Add a new "Install Conan dependencies" step immediately before the "Configure" step
3. For matrix workflows: capitalize the build_type with `sed 's/./\u&/'` because cmake uses lowercase (`debug`) but Conan requires title case (`Debug`)
4. Set `--output-folder` to match the path in CMakePresets.json (typically `build/$BUILD_TYPE`)
5. Apply this pattern to all three workflow files: `build-test.yml`, `static-analysis.yml`, `code-coverage.yml`
6. For coverage workflows, use the coverage preset output folder (e.g., `build/coverage`)

## Failed Attempts

| Attempt | What Was Tried | Why It Failed | Lesson Learned |
|---------|----------------|---------------|----------------|
| cmake --preset without conan | Ran `cmake --preset debug` directly after checkout | CMake exits: "Could not find toolchain file: build/debug/conan_toolchain.cmake" | conan_toolchain.cmake must be generated by `conan install` before cmake configure |
| Using lowercase build_type directly | `conan install . -s build_type="$BUILD_TYPE"` with `BUILD_TYPE=debug` | Conan rejects lowercase: expects `Debug` not `debug` | Capitalize with `sed 's/./\u&/'` before passing to `-s build_type=` |
| Only fixing build-test.yml | Fixed build-test.yml but left static-analysis.yml and code-coverage.yml | Static analysis and coverage workflows still fail at configure | All three workflow files need the conan install step |
