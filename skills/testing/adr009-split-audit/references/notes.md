# Session Notes: ADR-009 Split Audit

## Session Date
2026-03-15

## Issue
ProjectOdyssey #4241 — "Verify gradient-checking tests preserved in other ADR-009 splits"
Follow-up from #3444 which discovered that `test_backward.mojo` split had silently dropped
7 gradient tests.

## Context
ADR-009 requires splitting Mojo test files to >10 tests into multiple files to avoid a
Mojo v0.26.1 heap corruption bug (`libKGENCompilerRTShared.so` crashes). The split process
can silently drop tests if the splitter doesn't carefully track all function names.

## What Was Investigated

The issue plan (posted as a GitHub issue comment) was generated by a planning agent and
described 10 missing tests across two files. The actual audit found:

1. `test_backward_*.mojo`: All 21 original tests present. The plan was stale — earlier
   PRs had already restored the 7 gradient tests (commit 127d8e02 explicitly says
   "All 21 original test cases are preserved").

2. `test_gradient_checking_*.mojo`: All 16 original tests present. Commit d1420696
   "restore 3 missing tests" had already fixed the gradient_checking splits.

3. `test_extensor_unary_ops.mojo.DEPRECATED` → `test_extensor_neg_pos.mojo`:
   `test_unary_ops_preserve_shape` was missing. This was NOT in the issue plan
   (the plan only looked at backward/gradient_checking files). Discovered by expanding
   the audit to all `.DEPRECATED` files.

## Key Technical Details

- `.DEPRECATED` files are kept in the repo as permanent reference
- Some source files were deleted entirely (no `.DEPRECATED` left) — must use git history
- `test_backward.mojo` had no `.DEPRECATED` marker; found via `git log --diff-filter=D`
- CI uses wildcard `test_extensor_*.mojo` so no CI update needed for the fix
- The "Core Gradient" group in comprehensive-tests.yml uses explicit filenames — only
  that group needs updating when creating NEW split files (not needed here)

## Fix Applied

Added `test_unary_ops_preserve_shape` to `test_extensor_neg_pos.mojo`:

```mojo
fn test_unary_ops_preserve_shape() raises:
    """Test that unary operators preserve tensor shape."""
    var shape: List[Int] = [3, 4, 2]
    var a = zeros(shape, DType.float32)
    var neg_result = -a
    assert_equal(len(neg_result.shape()), 3)
    var pos_result = +a
    assert_equal(len(pos_result.shape()), 3)
```

PR: HomericIntelligence/ProjectOdyssey#4877

## Audit Script Used

```bash
for deprecated in tests/shared/core/*.DEPRECATED; do
    echo "=== $deprecated ==="
    grep "^fn test_" "$deprecated" | sed 's/fn //; s/(.*$//' | sort > /tmp/dep_tests.txt
    grep -h "^fn test_" tests/shared/core/*.mojo 2>/dev/null \
        | sed 's/fn //; s/(.*$//' | sort > /tmp/split_tests.txt
    missing=$(comm -23 /tmp/dep_tests.txt /tmp/split_tests.txt)
    if [ -z "$missing" ]; then
        echo "✓ All tests preserved"
    else
        echo "✗ MISSING: $missing"
    fi
done
```

## Lesson: Stale Issue Plans

Issue plans generated by planning agents reflect a point-in-time snapshot. Between plan
generation and implementation, other PRs may have already fixed some or all of the issues.
Always re-audit current state rather than blindly implementing the plan.
